# 五行生肖·遗物试炼

一个基于中国五行与十二生肖主题的单文件 HTML 小游戏原型，当前方向为 **Roguelike 单局构筑**。

## 当前开发状态

- 已完成：可游玩的单局流程（战斗/商店节点，共 8 节点：5 战斗 + 3 商店）。
- 已完成：五行相生相克参与结算。
- 已完成：10 个遗物（5 普通 + 3 稀有 + 2 传说）全部接入。
- 已完成：5 个敌人含独特技能（封印、护盾、复苏、宣言等）。
- 已完成：桌面端水平双栏布局（左栏转盘 + 右栏操作面板）。
- 已完成：开局三选一免费遗物。
- 已完成：生命损失时多重视觉效果（脉动、闪红、浮动文字）。
- 暂未实现：局外成长（元进度、永久天赋等）。
- 暂未实现：移动端响应式适配。

## 视觉设计要求

### 转盘视觉设计要求
- 外环套内环的圆形转盘设计
- 外环：生肖均匀分布在外圈，每个生肖使用不同颜色
- 内环：五行均匀分布在内圈，每个五行使用不同颜色（不使用图标）
- 中心区域：五行颜色直接从中心向外延伸，形成实心圆形
- 押注方式：点击圆盘对应区域进行押注（点击外环押注生肖，点击内环押注五行）
- 选中区域：通过高亮发光效果显示当前押注选择（不使用指针）
- 物理惯性效果：转盘启动时有加速过程，停止时有减速过程（缓动效果）
- 支持动态段数：龙门契移除元素/生肖后转盘自动调整
- 双仪盘阴面标记：☯ 符号 + 暗淡色区分阴阳

### 转盘技术实现要求
- 最终中奖结果必须与动画显示一致
- 使用 requestAnimationFrame 实现平滑动画
- 缓动函数：使用 ease-out 模拟物理减速
- 点击押注区域时，该区域高亮显示选中状态
- 段数可变（availableElems/availableZods），绘制/动画/点击均适配

## 文件结构

- `game.html`：完整游戏（UI、逻辑、样式，单文件自包含）。
- `prompt.md`：需求与状态文档（当前文件）。

## 如何运行

无需安装依赖，直接用浏览器打开：

1. 打开 `game.html`
2. 点击「新开一局」
3. 选择起始遗物（或跳过）
4. 按节点推进：战斗 → 商店 → 战斗 ...

## 核心玩法（当前版本）

### 1. 节点流程

- 固定节点序列：`battle → shop → battle → shop → battle → shop → battle → battle`
- 战斗胜利：弹出三选一遗物奖励。
- 商店节点：展示最多 3 个可购买遗物，可买多个，离开后前进。
- 开局前：三选一免费遗物（可跳过）。

### 2. 战斗规则

- 玩家每回合下注：`金额 + 五行 + 生肖`。
- 双环转盘（五行盘/生肖盘）同时旋转后结算。
- 结算包含：
  - 五行命中（×2.2）
  - 生肖命中（×4.0）
  - 双命中加成（×3.8）
  - 五行相生命中加成（×0.8）
  - 五行相克惩罚（−0.25×，可被遗物抵消/转化）

### 3. 资源系统

- 金币（初始 150）：下注与购买遗物消耗，战斗收益增加。
- 命元（初始 10）：战斗失败扣除，归零则结束。
- 护甲：可抵消部分失败伤害（由厚土印提供，每场胜利 +1）。

### 4. 敌人系统

| # | 名称 | HP | 护盾 | 技能 |
|---|---|---|---|---|
| 1 | 界门守卫 | 80 | 0 | 无 |
| 2 | 幽冥判官 | 150 | 0 | 幽冥判决（偶数回合亏损额外扣1命元） |
| 3 | 五行道士 | 250 | 0 | 元素封印（随机封印一相性，命中减半）+ 克制强化（相克惩罚×2） |
| 4 | 轮回使者 | 350 | 50 | 轮回护盾 + 轮回复苏（HP≤40%时自愈80HP，仅一次） |
| 5 | 大衍术士 | 480 | 0 | 大衍宣言（命中宣言五行伤害×2）+ 七星回煞（每3回合−30收益）+ 天机壁障（双命中加成减半） |

## 当前平衡参数（代码内）

- 基础倍率配置在 `game.html` 的 `MULT` 常量中：
  - `element: 2.2`（五行命中）
  - `zodiac: 4.0`（生肖命中）
  - `bothBonus: 3.8`（双命中额外加成）
  - `sheng: 0.8`（相生加成）
  - `kePenaltyRate: 0.25`（相克惩罚）
- 初始金币：150
- 初始命元 / 最大命元：10 / 10
- 默认下注额：20

## 技术实现要点

- 单文件 HTML 自包含（CSS + JS + HTML 全在一个文件中）。
- Canvas 绘制双环转盘，支持动态段数（龙门契移除后自适应）。
- 遗物获取时触发 `applyRelicOnAcquire()` 处理特殊初始化（龙门契移除、双仪盘阳面确定）。
- 游戏状态集中在 `gs` 对象（由 `makeState()` 创建）。
- 敌人由 `ENEMIES` 模板数组定义，通过 `initEnemy()` 实例化。
- 结算逻辑集中在 `settle()` 函数。

## 已知限制

- 当前只有前端本地逻辑，无存档/读档。
- 节点路线是固定的，尚未做随机地图分支。
- 无自动化测试与数值模拟脚本。
- 未做移动端响应式适配。
